## 🔍 IronClaw 概述

**定位**: OpenClaw 的安全增强替代方案，运行在 NEAR AI Cloud 的 TEE 加密 enclave 中

**核心卖点**:
- Secrets 永不暴露给 LLM (Encrypted Vault)
- 每个工具独立 Wasm 沙箱 (Tool Isolation)
- 内存级加密 (Trusted Execution Environment)
- Rust 实现 (内存安全)
- 网络端点白名单 (Allowlist)

**技术栈**:
- 语言: Rust (vs OpenClaw 的 TypeScript)
- 运行时: WASM 沙箱
- 加密: TEE (Trusted Execution Environment)
- 部署: NEAR AI Cloud (1-click)

## 💡 关键洞察

### 1. 安全模型的根本差异

**OpenClaw 的风险** (IronClaw 官方文档指出):
- Prompt injection 可直接 dump 所有 API keys
- 恶意 skill 可在社区传播 (数百个已被发现)
- 30,000+ 公开实例正在被攻击者利用

**量化交易场景的特殊风险**:
- 交易所 API keys → 可以交易、提现 → **资金损失**
- 数据库连接串 → 交易日志、绩效数据 → **合规风险**
- 风控参数 → 止损、仓位限制 → **策略被绕过**

### 2. IronClaw 的技术亮点

```rust
// 部署流程 (官方演示)
fn deploy(cfg: &Config) -> Result<()> {
    let tee = TeeEnclave::provision()?;          // TEE 创建
    tee.verify_memory_safety()?;                 // 内存安全检查
    let vault = Vault::seal(cfg)?;               // 密封凭证
    vault.bind_endpoints(&cfg.allowlist)?;      // 绑定端点白名单
    agent::spawn(tee, vault)                     // 启动
}
```

**Secret 注入机制**:
- Credentials 加密存储在 vault
- 只在网络边界注入到请求 (LLM 看不到明文)
- 仅对白名单端点生效

**Wasm 沙箱**:
- 每个工具独立容器
- 无文件系统访问
- 无未授权网络连接
- 严格的资源限制

## 📝 具体实施建议

### 1. Vault 模块设计 (借鉴 IronClaw)

```typescript
// src/core/vault.ts
class Vault {
  private secrets: Map<string, EncryptedSecret>;
  private masterKey: Uint8Array; // 从环境变量读取
  
  async store(key: string, value: string, scope: SecretScope): Promise<void> {
    const encrypted = await this.encrypt(value);
    this.secrets.set(key, { ciphertext: encrypted, scope });
  }
  
  async inject(key: string, request: Request): Promise<void> {
    const secret = this.secrets.get(key);
    if (!secret) throw new Error(`Secret ${key} not found`);
    
    // 检查 scope 是否允许此 endpoint
    if (!secret.scope.allows(request.url)) {
      throw new Error(`Endpoint not allowed for ${key}`);
    }
    
    // 注入请求头
    request.headers.set('Authorization', await this.decrypt(secret.ciphertext));
  }
}
```

### 2. Allowlist 机制

```typescript
// src/core/allowlist.ts
interface EndpointRule {
  pattern: string;      // URL pattern (wildcard)
  methods: string[];    // GET, POST, etc.
  rateLimit?: number;   // optional rate limit
}

class Allowlist {
  constructor(private rules: EndpointRule[]) {}
  
  permits(url: string, method: string): boolean {
    return this.rules.some(rule => 
      this.match(rule.pattern, url) && 
      rule.methods.includes(method)
    );
  }
}
```

**Skill 声明示例**:
```yaml
skill.yaml:
  name: "Binance Trade Executor"
  endpoints:
    - pattern: "https://api.binance.com/api/v3/order"
      methods: ["POST"]
    - pattern: "https://api.binance.com/api/v3/account"
      methods: ["GET"]
```

### 3. Worker 隔离

- 每个策略在独立 Node.js worker 线程中执行
- 通过 IPC 通信，不共享内存
- 超时自动 kill (防无限循环)
- 资源限制 (CPU, memory)

## 🔮 未来展望

IronClaw 代表了 AI Agent 安全的重要演进方向。对于量化交易这种高敏感场景，**安全不应是事后补救，而必须是设计基石**。

无论最终选择 OpenClaw 自托管还是 IronClaw 云原生，以下原则不变：

1. **零信任**: LLM 永远不应看到明文 credentials

3. **纵深防御**: 多层安全 (vault + allowlist + sandbox + 审计)
4. **可验证**: 所有安全声称都能被验证 (audit trail)
